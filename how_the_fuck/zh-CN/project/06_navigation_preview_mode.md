# Navigation Preview 模式

## 目标

这个模式不是正式导航引擎，而是：

**先把“手机里导航时看起来应该像什么”做出来。**

这样你可以先调视觉、层级、路线强调方式，再决定后续产品细节。

---

## 这一版做了什么

1. 地图 review 页面支持点击节点
2. 第一次点击节点：
   作为当前起点
3. 第二次点击节点：
   作为目的地，并进入导航预览
4. 导航预览状态下：
   - 高亮路径
   - 顶部导航卡片
   - 底部目的地栏
   - 背景降噪
   - 自动把起点放到更接近屏幕下半区的位置

---

## 为什么这样做

这是比较符合手机导航常见做法的第一步：

1. 用户先选当前位置
2. 再选目标点
3. 屏幕优先强调“前方路线”，而不是整个底图
4. 当前屏幕保持路线可见，同时仍能知道自己在哪里

---

## 这一版不是最终版

还没做的内容：

1. 真正的逐步转向提示
2. 路口级别指引
3. 当前实时位置更新
4. 更强的目的地卡片
5. 分区导航和楼层切换

---

## 现在你主要要调什么

1. 路线颜色够不够醒目
2. 背景降噪是否太重或太轻
3. 顶部卡片和底部栏是否挡住关键信息
4. 起点默认落点是否合理
5. 节点标签是否太多
6. 当前手机缩放倍率是否适合导航状态
