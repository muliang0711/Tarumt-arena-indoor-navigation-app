# 当前决策记录

## 这份文档的目的

这份文档只记录我们**当前已经决定**的方案。

它不是 brainstorming，也不是长期愿景。

它的作用是：

- 防止后面继续反复犹豫
- 让接下来的实现有一个明确方向
- 把今天讨论出来的内容固定下来

---

## 今天决定改什么

我们决定先把现在的方案直接做起来，不再继续停留在抽象讨论阶段。

当前的重点不是重新设计一套全新的系统，而是：

**把现有的 `Village Map Editor` 扩展成一个可以制作导航地图的编辑器。**

---

## 决策 1：不再把它当成单纯画图工具

`Village Map Editor` 不再只是做视觉地图。

我们决定把它扩展成一个**两层编辑系统**。

### Layer 1：Visual Layer

这个 layer 只负责画图：

- 摆放 PNG 组件
- 还原学校平面布局
- 处理房间、走廊、楼梯、电梯、墙等视觉元素

这个 layer 的目标是：

- 把学校地图画出来
- 尽量接近真实布局
- 但不要求所有视觉尺寸严格 1:1

### Layer 2：Metadata Layer

这个 layer 负责导航相关信息：

- walkable / blocked
- room entrance anchor
- stairs anchor
- elevator anchor
- destination metadata
- 后续可能的连接关系

这个 layer 的目标是：

- 让地图从“图片”变成“可导航地图”

---

## 决策 2：导航底层先用 Grid

我们决定：

**第一版导航系统先做 Grid，不先做 Waypoint。**

原因：

- 现在的编辑器本来就是 tile/grid 思维
- Grid 更容易和现有地图编辑器结合
- 逻辑更直观
- 更适合先把系统跑通

### 当前结论

- 先做 `grid-based navigation`
- 后面再评估是否升级成 `waypoint-based navigation`

### 当前不做的事

- 不先手工搭 waypoint graph
- 不先维护复杂 node-edge 系统
- 不先优化成更抽象的高层路径图

---

## 决策 3：房间/建筑的导航目标不是整块区域，而是入口锚点

我们已经明确：

- 大房间、大建筑本体是一个区域
- 这个区域大部分情况下是 blocked
- 导航不会直接走到房间内部某个 tile
- 导航目标应该是门口前面的入口 anchor

### 这意味着什么

以后在 Metadata Layer 里，要明确给房间标：

- footprint
- blocked area
- entrance anchor

例如：

- `A401` 房间本体是一大片区域
- `A401_entrance` 才是导航终点

---

## 决策 4：地图制作采用“两次编辑”的工作流

我们决定，地图不是一次编辑完成，而是分两次做。

### 第一次编辑：先把视觉地图铺完

这一阶段只关心：

- 地图组件摆放
- 房间位置
- 走廊
- 楼梯
- 电梯
- 墙和门

输出：

- 一张可视地图
- 一份 visual placement 数据

### 第二次编辑：再给地图补 metadata

这一阶段才开始处理导航逻辑：

- 哪些区域可走
- 哪些区域 blocked
- 哪些点是房间入口
- 哪些点是楼梯/电梯连接点

输出：

- 一份可导航的逻辑地图

---

## 决策 5：长度采用方案 C

我们已经决定：

**真实长度不强制映射到视觉上的 1:1 比例。**

当前采用的方案是：

## 方案 C

### 视觉层

- 地图可以是假比例
- 不需要所有走廊长度按真实长度严格拉伸
- 只要空间关系和连通关系正确即可

### 导航层

- 使用真实长度作为导航权重
- 路径计算、距离估算、后续 ETA 都基于真实长度

### 核心思想

不是把真实长度映射成“像素长度”，
而是把真实长度映射成“导航代价 / 权重”。

---

## 这意味着我们现在不做什么

当前明确不做：

- 不强迫整张地图视觉 1:1 还原
- 不为了真实米数去重新拉伸每个 PNG 尺寸
- 不为了长度问题卡住地图制作

当前明确要做：

- 先把地图逻辑跑通
- 后面再把真实长度接进导航权重层

---

## 当前产品方向

所以现在的产品方向已经明确为：

### Step 1

把 `Village Map Editor` 从单层画图工具，扩展成：

- Visual Layer
- Metadata Layer

### Step 2

先完成 grid-based 的导航地图制作能力：

- 视觉地图
- blocked / walkable
- anchors

### Step 3

长度使用方案 C：

- 视觉不做严格 1:1
- 导航权重使用真实长度

### Step 4

等第一版跑通以后，再决定是否升级 waypoint 系统

---

## 当前最终结论

我们今天已经决定：

1. `Village Map Editor` 要扩展成双层编辑器，而不是只画图
2. 第一版导航底层先做 Grid
3. 房间和建筑的导航目标使用 entrance anchor
4. 地图采用两次编辑流程：先视觉、后 metadata
5. 长度采用方案 C：视觉假比例，导航真实权重
6. Waypoint 不是现在做，而是未来升级选项

---

## 一句话版本

**现在先把 Village Map Editor 做成一个“能画图 + 能标导航 metadata”的 Grid 地图编辑器；视觉长度不强求 1:1，真实长度以后作为导航权重接入。**
