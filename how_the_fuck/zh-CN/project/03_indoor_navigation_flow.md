# 学校室内导航到底怎么跑起来

## 你现在困惑的点

你现在的困惑其实很正常，因为你脑中混在一起的是两件不同的事：

1. **定位系统**
   负责回答：`用户现在在哪里？`

2. **导航系统**
   负责回答：`从用户当前位置到目标地点应该怎么走？`

你前面会觉得像是“两套系统”，是因为它本来就是两套子系统，只是最后要接在一起。

---

## 先用一句话讲清楚整个流程

在你的学校四层楼场景里，完整流程应该是：

**Wi-Fi fingerprinting 估计用户位置 -> 把这个位置映射到 2D 地图上的某个可行走点 -> 根据目标房间计算跨楼层路径 -> 在 2D 地图上显示路线并持续更新**

所以真正的系统链路不是“做地图”这么简单，而是：

1. 地图建模
2. 室内定位
3. 位置映射
4. 路径规划
5. 导航展示

---

## 这五个部分分别是什么

## 1. 地图建模

地图不是单纯一张 PNG。

对导航来说，地图至少要有这些数据：

- 每层楼的平面图
- 可走区域
- 不可走区域
- 房间/教室/办公室的位置
- 楼梯和电梯的位置
- 各楼层之间如何连接

你可以把它理解成：

- **视觉地图**：给人看
- **逻辑地图**：给算法算路

视觉地图可以很漂亮，但导航真正依赖的是逻辑地图。

---

## 2. 室内定位

你的想法是用 **Wi-Fi fingerprinting**，这个方向是合理的。

它的基本思想是：

- 在楼里很多采样点，提前采集 Wi-Fi 信号强度
- 每个采样点都对应一个位置标签
- 用户运行时，手机再扫一次当前 Wi-Fi
- 系统拿当前扫描结果去和数据库里历史 fingerprint 比较
- 找到最接近的点，估计“用户在这里”

### 它分两阶段

### A. Offline phase

离线建库阶段：

- 在 1 楼、2 楼、3 楼、4 楼的很多采样点收集 RSSI
- 每个采样点保存：
  - 楼层
  - 坐标
  - 多个 AP 的信号强度

### B. Online phase

在线定位阶段：

- 用户手机扫描当前 Wi-Fi
- 输入定位模型
- 输出一个预测位置

这个输出通常不是“精确到门把手”，而是：

- 某一层楼
- 某个区域
- 某个采样点附近

所以它本身就带误差。

---

## 3. 位置映射

这一步很重要，也很容易被忽略。

Wi-Fi fingerprinting 给你的结果，不应该直接当成“用户已经在导航路径上”。

因为定位结果可能会：

- 落在墙里
- 落在房间里
- 落在不可走区域
- 落在走廊旁边一点点

所以你需要做一层 **map matching**：

- 先得到预测坐标
- 再把它“吸附”到最近的可行走点上

这一步的作用是：

- 让用户位置看起来更稳定
- 避免导航从墙里面开始
- 避免路径规划报错

你可以理解成：

**定位系统给“原始位置”，地图系统给“修正后位置”**

---

## 4. 路径规划

这个才是导航系统的核心。

当系统知道：

- 用户当前在哪个点
- 用户要去哪一个教室/办公室

就可以开始算路径。

### 你这里最适合的做法

对于学校四层楼，我建议你先把路径规划拆成两层：

### A. 单层内路径

每层楼自己有一张逻辑图：

- 走廊可以走
- 房间内部默认不可走
- 墙不可走
- 门口是进入目标房间的锚点

然后在该层楼上做 A* 或 Dijkstra。

### B. 跨楼层路径

楼梯和电梯作为“连接节点”：

- `stairs_1F_A <-> stairs_2F_A`
- `stairs_2F_A <-> stairs_3F_A`
- `lift_1F <-> lift_2F <-> lift_3F <-> lift_4F`

如果起点和终点不在同一层：

1. 先算当前楼层到最近合适楼梯/电梯的路径
2. 再经过跨楼层连接
3. 最后算目标楼层到目标房间入口的路径

这就是多层导航的本质。

---

## 5. 导航展示

最后才是用户看到的部分。

系统把算好的路径画在 2D 地图上：

- 当前用户位置
- 目标位置
- 推荐路径
- 当前所在楼层
- 到下一段的提示

跨楼层时可以这样做：

- 当前显示“先走到 2 楼楼梯 A”
- 到达后切到下一层地图
- 再继续显示下一段路径

---

## 你现在脑中那“两套系统”到底是什么

你可以把它们明确分开：

## 系统 A：定位系统

输入：

- 手机扫描到的 Wi-Fi RSSI

输出：

- 用户在 `floor = 2`
- 用户大概在 `(x, y)`

这个系统只负责“找人”。

## 系统 B：导航系统

输入：

- 当前楼层和位置
- 目标房间
- 地图逻辑结构

输出：

- 一条可走的路径

这个系统只负责“带路”。

两者通过“当前位置”这个接口连接起来。

---

## 你项目里的完整 end-to-end 流程

假设一个学生要从当前所在位置去 `Lab 3A`：

1. 用户打开 App
2. 手机扫描当前 Wi-Fi
3. fingerprinting 模型估计用户在 `2F` 的某个位置
4. map matching 把该位置吸附到 `2F` 走廊上的最近可行走点
5. 用户选择目标 `Lab 3A`
6. 系统查出 `Lab 3A` 的入口点在 `3F`
7. 路径规划模块发现起点和终点不在同一层
8. 先算 `2F 当前点 -> 2F 楼梯/电梯`
9. 再经过跨楼层连接到 `3F`
10. 再算 `3F 楼梯/电梯 -> Lab 3A 入口`
11. 把整条路径分段显示在地图上
12. 用户移动时重新扫描 Wi-Fi
13. 如果位置偏移较大，就重新规划路径

这就是完整的导航闭环。

## 你现在最容易卡住的原因

你现在会迷茫，不是因为你不会，而是因为你脑中同时在做四件事：

- 做地图美术
- 做地图逻辑
- 做定位
- 做导航算法

这四件事如果同时想，会非常乱。

正确做法是按顺序拆开。

---

## 你现在最应该先想清楚的，不是“全部”

先把问题拆成下面这几个最小问题：

### Q1. 用户的位置最终在地图上用什么表示？

答案：

- `floor`
- `x`
- `y`

再加一个“映射后的 walkable node”

### Q2. 目标地点在系统里用什么表示？

答案：

- 不是房间的整块区域
- 而是一个目标入口点或者目标节点

例如：

- `Room A101 -> entrance node`
- `Lab 3A -> entrance node`

### Q3. 地图应该保存什么？

答案：

- 每层楼的 walkable graph 或 walkable grid
- 房间入口点
- 楼梯/电梯连接关系

### Q4. 跨楼层怎么做？

答案：

- 把楼梯和电梯当成连接节点
- 用它们把四层图串起来

### Q5. Wi-Fi fingerprinting 和导航怎么接？

答案：

- fingerprinting 输出位置
- 位置映射模块把它对齐到地图逻辑点
- 导航模块从这个逻辑点开始算路

## 如果你要做论文/原型，最小可行版本应该是什么

我建议你先做一个 **MVP**，不要一开始追求全部真实细节。

### MVP Version 1

- 只做一栋楼
- 四层
- 每层地图是逻辑图，不一定要超精美
- 用 Wi-Fi fingerprinting 得到楼层 + 大致坐标
- 用 map matching 把坐标吸附到走廊
- 用 A* 做跨楼层导航
- 在 2D 地图上显示路径

### Version 1 先不要做太复杂的东西

- 不要先做 AR
- 不要先做实时转角提示
- 不要先做过度精细的室内姿态估计
- 不要先做太复杂的用户行为预测

先证明：

**在学校四层楼中，系统能定位、能选目标、能算路、能显示路径**

这已经是一个完整系统了。

## 一个最清晰的系统架构图

你可以把整体架构理解成这样：

### 1. Data layer

- 楼层地图
- 房间信息
- 楼梯/电梯连接
- Wi-Fi fingerprint database

### 2. Localization layer

- 扫描 Wi-Fi
- 指纹匹配/分类/回归
- 输出用户楼层和坐标

### 3. Mapping layer

- 把原始定位结果映射到地图逻辑点

### 4. Navigation layer

- 接收起点和终点
- 跨楼层路径规划
- 输出 route

### 5. UI layer

- 2D map
- user marker
- route polyline
- floor switching

---

## 你现在该怎么落地，不要再乱想

建议你按下面顺序做：

### Step 1. 先定义地图逻辑结构

不是先画得多漂亮，而是先定义：

- 每层有哪些可走路径
- 哪些点是房间入口
- 哪些点是楼梯/电梯

### Step 2. 先不管 Wi-Fi，手动给一个起点测试导航

例如：

- 起点：`2F corridor node 12`
- 终点：`3F Lab 3A entrance`

先证明路径规划是通的。

### Step 3. 再把 Wi-Fi fingerprinting 的输出接进来

让定位模块输出：

- 当前楼层
- 当前坐标

然后接到导航模块。

### Step 4. 再做位置修正

把定位点吸附到走廊图上。

### Step 5. 最后再做显示优化

- UI 美化
- 路径高亮
- 指示文字
- 切楼层体验

## 我对你这个项目的直接判断

你的项目不是“做一张地图”。

你的项目真正要做的是：

**一个基于 Wi-Fi fingerprinting 的多楼层室内定位与路径导航系统**

其中地图只是其中一层。

你现在最重要的认知转换是：

- PNG 地图只是表现层
- 导航真正依赖的是逻辑层
- Wi-Fi fingerprinting 解决“我在哪”
- pathfinding 解决“我怎么去”

当你把这两个问题分开，你就不会乱。

## 对这四个问题的简短答案

### 答案 1

navigation 的流程是：

**定位 -> 映射 -> 选目标 -> 算路 -> 显示路径 -> 持续更新**

### 答案 2

确实是两套系统：

- 定位系统
- 导航系统

它们通过“当前位置”连接。

### 答案 3

Wi-Fi fingerprinting 负责给出楼层和粗略坐标，2D map 负责承载逻辑地图和显示路线。

### 答案 4

先做：

1. 逻辑地图
2. 单层/跨层路径规划
3. 再接 Wi-Fi 定位
4. 最后做 UI 和美化

你现在不要问自己：

`我怎么一次做完整个系统？`

`如果先不管 Wi-Fi，我能不能先让四层地图自己算出一条正确路径？`

如果这个答案是 yes，再把定位系统接上去，整个项目就会清楚很多。
