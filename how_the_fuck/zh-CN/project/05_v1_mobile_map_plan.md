# 手机地图 V1 方案

## 目标

第一版不要追求“整张图一次看清全部细节”。

第一版只解决两个问题：

1. 用户在手机上能看懂当前楼层的大结构。
2. 用户在手机上能放大后仍然不迷路，并能看见关键语义点。

---

## 第一版范围

## 必做

1. `Overview / Readable / Detail` 三档视图
2. `Readable` 作为地图评审的默认状态
3. `Minimap + viewport box`
4. 复用 editor 现有节点元数据做第一层语义 overlay
5. 支持起点 / 终点选择，并高亮一条基础路线

## 暂不做

1. 完整房间分区浏览
2. 大规模 UI 重设计
3. 丰富图标系统
4. 高级无障碍主题切换
5. 真正产品级路径指引动画

---

## 为什么这样切

### 不先重画地图

因为当前最大问题不是美术，而是手机上的信息读取方式。

### 不先做复杂 overlay 系统

因为 editor 已经有 `nodes` 和 `links`，第一版直接复用最稳。

### 不先做分区系统

因为分区是第二阶段优化，不是第一阶段的可用性底线。

---

## 本次已开始实现的内容

已经落地到 `village_map_editor` 的第一版功能：

1. 地图快照发送到 phone reviewer 时，会同时带上：
   - `nodes`
   - `links`
   - `mapWidth`
   - `mapHeight`
   - `tileSize`
   - `exportScale`
2. phone reviewer 新增：
   - 语义 overlay 面板
   - 节点显示开关
   - 标签显示开关
   - 起点选择
   - 终点选择
   - 清除路线
3. reviewer 会根据 editor 里的图结构做基础最短路高亮
4. reviewer 对地图模式默认使用更偏 `Readable` 的倍率

---

## 第一版的工程定义

如果一个版本满足以下条件，就算 V1 达标：

1. 用户能在手机里一眼看清整层的大概结构
2. 用户能一键切到更清晰的阅读态
3. 放大后不会失去方向
4. 用户能看到关键节点标签
5. 用户能在两个节点之间看到一条高亮路径

---

## 第二版再做什么

1. 房间名标签体系正式化
2. `toilet / stairs / elevator / destination` 图标语义分层
3. 分区切换
4. 当前定位与目的地状态
5. 更强的路径强调和背景降噪

---

## 结论

V1 的核心不是“把地图做得更华丽”，而是：

**先把手机端的阅读路径做出来。**

也就是：

`看全局 -> 放大 -> 不迷路 -> 看语义点 -> 看路线`
