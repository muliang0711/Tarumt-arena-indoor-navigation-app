# Map Navigation UI Design

## Goal

Add an isolated development navigation UI that derives its origin from Bob's actual actor spawn state, selects Nodes 2-4, calculates graph routes, highlights every route edge, and visualizes movement constraints without changing sensor-driven movement.

## Architecture

The normalized map contract will expose typed route edges. A pure debugger navigation model will own destination selection, Dijkstra path calculation, route replacement/clearing, and overlay projection. React Native components under `src/mapEngine/debugger/` will render controls and visual layers. `ArenaMapEngineView` remains the composition owner and passes the same normalized route graph and `MovementConstraintMapInput` used by movement.

The navigation origin is `startingActor.nodeId`, not a visual constant. Bob's world position remains owned by `MovementRuntime`; selecting or calculating a route never writes actor or runtime state.

## Route Graph and Pathfinding

Route nodes and edges are normalized from the existing map schema. Enabled edges participate in routing, and bidirectional edges create reverse adjacency entries. Dijkstra uses edge `weight`, falling back to `distance_m` and then Euclidean node distance.

Expected default-map paths are:

- Node 1 → Node 2: `node_1`, `node_2`
- Node 1 → Node 3: `node_1`, `node_2`, `node_3`
- Node 1 → Node 4: `node_1`, `node_2`, `node_4`

Node 1 → Node 4 therefore contains the vertical Node 1-to-Node 2 segment followed by the horizontal Node 2-to-Node 4 corridor edge. Node 2 is the L-shaped turn. The direct Node 2-to-Node 4 edge is preferred over the equal-weight multi-edge alternative through Node 3 because it uses fewer graph edges.

Destination selection does not move Bob. If no route is displayed, selection only changes the selected marker. If a route is displayed, changing destination recalculates and replaces it. Calculation failures produce an explicit no-route state.

## Rendering and Controls

Debugger components provide:

- destination buttons for Nodes 2, 3, and 4;
- `Calculate route`;
- `Clear route`;
- `Show/Hide unwalkable`;
- a start marker derived from the actor spawn node;
- a selected-destination marker;
- highlighted graph edge segments;
- the existing Bob actor marker/sprite;
- a semi-transparent red constraint overlay.

Styles remain distinct: green start, orange destination, blue route, red constraints, and the existing Bob sprite.

## Constraint Overlay

The overlay receives the exact `MovementConstraintMapInput` passed to `MovementRuntime`. It renders:

- blocked polygons from `blockedAreas`;
- wall boundaries from `walls`;
- practical outside-walkable coverage generated by sampling map tile centers through `createMovementConstraintProvider(...).isWalkable`.

Adjacent non-walkable cells are merged into row rectangles to avoid thousands of React Native views. The model retains the original normalized walkable, blocked, and wall geometry for tests and diagnostics. No collision rules are weakened or duplicated.

## Isolation

All temporary navigation state, pathfinding, controls, and overlay rendering stay behind `src/mapEngine/debugger/index.ts`. Core rendering receives overlays through the existing `ArenaMapView.renderOverlay` hook. No sensor adapter, movement estimate, particle filter, or constraint behavior changes.

## Error Handling

- Missing destination nodes are disabled/unavailable.
- Invalid or disconnected graph data returns `no-route`.
- Missing route origin returns `no-route`.
- Empty walkable geometry follows the constraint provider's current unrestricted semantics; only explicit blocked areas and walls are rendered.
- Route calculation is immutable and cannot mutate the route graph or Bob.

## Verification

Pure tests cover initial origin, destination availability, all expected graph paths, the Node 4 turn, replacement and clearing, no-route behavior, shared constraint geometry, and Bob-position immutability. Contract tests cover edge normalization. Final verification runs typecheck, the complete test suite, and a web UI smoke test with screenshots.
